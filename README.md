# Claude Code Learning Project 🚀

Welcome! This is a **beginner-friendly learning project** to practice fixing bugs using Claude Code (an AI coding assistant).

---

## 📚 What is This Project?

This is a **practice sandbox** where you'll:
- Learn how to use Claude Code (AI-powered coding assistant)
- Find bugs in code and fix them
- Learn Git and GitHub workflows
- Create Pull Requests (PRs) 
- Manage projects with task tracking tools like Asana
- Practice real-world developer skills

**Think of it like:** A coding gym where you practice with small challenges to build your skills.

---

## 🎯 Who is This For?

- ✅ People who just finished **Claude Code 101**
- ✅ Beginners learning to code
- ✅ People wanting to learn Git/GitHub
- ✅ Anyone wanting to practice with AI-assisted coding
- ✅ Students, hobbyists, or career changers

---

## 💡 What You'll Learn

1. **How to use Claude Code** - Prompting AI to help you write and fix code
2. **Git basics** - Clone, branch, commit, push (don't worry, we'll explain!)
3. **GitHub Pull Requests** - How to submit code changes
4. **Fixing bugs** - Finding and solving problems in code
5. **Teamwork skills** - Writing clear descriptions, documenting changes
6. **Project management** - Tracking work with Asana tickets

---

## 🐛 The Bugs (Challenges)

This project has **1-2 intentional bugs** planted in the code. Your job is to:

1. **Find** the bug (read the code, understand what's wrong)
2. **Fix** the bug (write corrected code or use Claude Code to help)
3. **Test** it (make sure it works)
4. **Document** it (write a clear Pull Request)
5. **Track** it (update Asana ticket)

Each bug is designed to teach you something new! 🎓

## 🛠️ How to Get Started

### What You Need
- A GitHub account (free at github.com)
- Claude Code installed on your computer
- Git installed (free at git-scm.com)
- An Asana account (free at asana.com)
- A code editor (VS Code is popular and free)

### Step 1: Copy This Project to Your Computer

```bash
git clone https://github.com/YOUR-USERNAME/claude-code-learning.git
cd claude-code-learning
```

*(Replace `YOUR-USERNAME` with your actual GitHub username)*

### Step 2: Read the Code

Open the code files and try to understand what the program does. Don't worry if it's confusing at first - that's normal!

### Step 3: Find the Bugs

Look for comments that say `TODO: Fix this bug` or check the **BUGS.md** file for hints.

### Step 4: Use Claude Code to Fix It

```bash
claude code "I found a bug in [filename]. The problem is [description]. Can you help me fix it?"
```

### Step 5: Test Your Fix

Run the code to make sure it works now!

```bash
python main.py
```

### Step 6: Create a Pull Request (PR)

- Create a new branch: `git checkout -b fix/bug-description`
- Commit your changes: `git commit -m "Fix: bug description"`
- Push to GitHub: `git push origin fix/bug-description`
- Go to GitHub.com and create a Pull Request with a clear description

### Step 7: Update Your Asana Ticket

Add a comment with the PR link and move the ticket to "Done" ✅

## 📝 Quick Explanation of Key Terms

**Bug** - A mistake in code that makes the program not work correctly

**Git** - A tool that tracks changes to your code (like save points in a video game)

**Pull Request (PR)** - A formal way to say "Hey, I fixed something! Can you check my fix?"

**Clone** - Making a copy of a GitHub project on your computer

**Branch** - Like creating a different version of your code to work on without messing up the original

**Commit** - Saving your changes to Git with a message explaining what you did

**Asana** - A tool for organizing tasks and tracking work

**Claude Code** - An AI assistant that helps you write and fix code

## ✨ Tips for Success

🎯 **Start small** - Fix one bug at a time
